# New Repository Checklist

After creating a new repository from this template, review the following:

## Basic setup
- [ ] Rename the repository
- [ ] Update `README.md`
- [ ] Fill `docs/project-context.md`
- [ ] Review `CLAUDE.md`
- [ ] Review `AGENTS.md`
- [ ] Review `docs/process.md`

## Workflow setup
- [ ] Confirm issue templates fit this project
- [ ] Confirm PR template fits this project
- [ ] Decide whether blank issues should be enabled
- [ ] Define initial labels
- [ ] Define initial milestone strategy
- [ ] Review `docs/memory/MEMORY.md` and decide which lessons are shared vs project-local
- [ ] If this repo should sync shared files back to the template, copy and customize `.template/sync-template.yml.example`

## Branch / repository settings
- [ ] Confirm default branch name
- [ ] Configure branch protection if needed
- [ ] Confirm review policy
- [ ] Confirm merge policy

## Project-specific additions
- [ ] Add stack-specific files
- [ ] Add runtime / framework config
- [ ] Add test config
- [ ] Add CI config if needed
- [ ] Add deployment config if needed
- [ ] Add `.env.example` if needed

## Cleanup
- [ ] Remove template-only placeholder text
- [ ] Remove sections that do not apply
- [ ] Ensure no misleading default content remains
